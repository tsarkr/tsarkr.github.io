The Core Challenge: Historical Orthography

  • Mixed Script Complexity The raw records are not simple texts. They are a chaotic mixture of Hangul, traditional Hanja, and early 20th-century Japanese.
  • Japanese Colonial Kanji The extensive use of Japanese-style Hanja variants (Shinjitai) from the 1910s requires us to completely redefine orthographic and transcription rules.
  • Standardization First Before applying TEI tags or a Graph DB, a robust NLP preprocessing pipeline must be engineered to normalize these diverse historical characters.

Paradigm Shift: Hierarchy to Network

  • The Vertical Limit of TEI TEI (XML) is inherently hierarchical and document-centric. It is excellent for preserving text structure but struggles to map horizontal relationships that cross multiple documents.
  • The Horizontal Power of Ontology Ontologies (CIDOC-CRM) and Graph Databases break document boundaries. They focus on entities—creating a horizontal, interconnected web of real-world history.
  • The DH Bridge My pipeline solves this by transforming the rigid vertical hierarchies of TEI texts into a dynamic horizontal knowledge network.

Beyond Search: Generative AI (RAG)

  • Keyword Limitation Traditional engines only match words, missing complex multi-layered networks across records.
  • RAG Architecture Layering a Retrieval-Augmented Generation (RAG) system over the Graph DB allows for complex natural language queries.
  • Hallucination-Free LLMs generate answers strictly grounded in the TEI-tagged ontological graph, ensuring absolute historical accuracy.

Technical Pipeline & Implementation

  • The 'Docent' Pipeline Engineered to parse PostgreSQL TEI-tagged columns (`<persName>`, `<placeName>`) and map relationships.
  • Neo4j Graph Database Transforming static text into a high-performance, queryable network of interconnected historical events via Python.
  • Streamlit UI & RAG Providing an interactive interface for graph visualization and AI-driven historical interpretation (docent).

Scaling to Global Archives: EEBO-TCP

  • The Ultimate Material EEBO-TCP contains 70,000+ early English books perfectly encoded in TEI XML. However, it currently functions mostly as a static text-search archive.
  • The Perfect Engine My docent pipeline is specifically built to ingest TEI XML (`<persName>`, `<placeName>`) and automatically generate Neo4j relationships.
  • Immediate Synergy This pipeline can instantly transform EEBO-TCP from a static text repository into a dynamic, AI-queryable Knowledge Graph of 15th-17th century intellectual history.
